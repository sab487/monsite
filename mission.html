01Analyse & Contexte
1.1 — Situation initiale : gestion locale non maîtrisée

Le Campus Numérique gère 200 étudiants et une équipe de formateurs répartis sur plusieurs salles de formation équipées de postes Windows 10/11 Pro. Avant l'intervention, chaque poste fonctionnait en mode workgroup (groupe de travail) : les comptes utilisateurs étaient créés localement, machine par machine, sans aucune supervision centralisée.

 Risques critiques identifiés
 Prolifération comptes locaux
Environnement vierge à chaque changement de poste
 Absence de contrôle
Accès libre au Panneau de config, terminal, logiciels
 Stockage anarchique
Travaux sur postes locaux — risque permanent de perte
 Charge admin colossale
Modification politique = intervention sur 200 postes
1.2 — Solution retenue : Active Directory + GPO

L'installation du rôle Active Directory Domain Services (AD DS) sur un serveur dédié sous Windows Server 2022, promu en contrôleur de domaine (DC) du domaine campus.lan. Les Stratégies de Groupe (GPO) appliquent automatiquement les restrictions pédagogiques à la connexion.

 Bénéfices de la solution AD DS
Authentification centralisée — un seul compte AD par utilisateur, valable sur tous les postes du domaine.
GPO par héritage OU — restrictions appliquées automatiquement à chaque objet de l'OU concernée.
Lecteur réseau mappé — dossier Devoirs accessible depuis n'importe quel poste via le lecteur S:.
Séparation des droits — formateurs avec héritage GPO désactivé, aucune restriction appliquée.
1.3 — Architecture du domaine campus.lan

02Conception de l'arborescence AD
2.1 — Arborescence OU du domaine campus.lan

L'arborescence Active Directory est conçue selon le principe de séparation des responsabilités. Chaque Unité d'Organisation (OU) regroupe des objets de même nature, permettant l'application ciblée des GPO par héritage.

ADUC — Arborescence campus.lan
🌐 campus.lan (DC=campus, DC=lan)
├── OU=ETUDIANTS ← GPO_RESTRICTIONS appliquée
├── Chevalier.Thomas
├── Martin.Alice
├── Dubois.Bob
└── ... 197 autres comptes étudiants
├── OU=FORMATEURS ← Héritage GPO désactivé
├── Dupont.Prof
└── ... 50 comptes formateurs
└── OU=ADMINISTRATION
└── admin.campus
2.2 — Plan des GPO et périmètre d'application

Chaque GPO est liée à une OU spécifique. Le mécanisme d'héritage GPO garantit que toute politique appliquée à OU=ETUDIANTS est automatiquement héritée par les comptes qu'elle contient.

Nom GPO OU liée Paramètre Valeur Statut
GPO_FOND_ECRAN OU=Etudiants Fond d'écran imposé \\SRVAD\sysvol\fond_campus.jpg Activé
GPO_RESTRICTIONS OU=Etudiants Masquer Panneau de configuration NoControlPanel = 1 Activé
GPO_RESTRICTIONS OU=Etudiants Accès invite de commandes DisableCMD = 1 Bloqué
GPO_LECTEUR_DEVOIRS OU=Etudiants Mappage lecteur réseau S: \\SRV-AD\Devoirs Activé
GPO_FORMATEURS OU=Formateurs Droits étendus (pas de restrictions) Héritage désactivé Exempt

03Mise en œuvre technique
3.1 — Installation du rôle AD DS et promotion en DC

Le rôle Active Directory Domain Services est installé depuis le Gestionnaire de serveur via « Ajouter des rôles et fonctionnalités ». L'assistant de promotion configure le serveur en tant que nouveau contrôleur de domaine pour une nouvelle forêt campus.lan. Niveau fonctionnel fixé à Windows Server 2016.

PowerShell Installation AD DS et promotion DC
# Installation du rôle AD DS
PS C:\> Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools
# Promotion en contrôleur de domaine — Nouvelle forêt
PS C:\> Install-ADDSForest `
-DomainName "campus.lan" `
-DomainNetbiosName "CAMPUS" `
-ForestMode WinThreshold `
-DomainMode WinThreshold `
-InstallDns `
-Force
# → Le serveur redémarre automatiquement après promotion
WARNING: Windows Server will reboot after installation
✓ ADDSFOREST : La forêt campus.lan a été créée avec succès.
3.2 — Création des OUs et des comptes utilisateurs

Depuis la console Utilisateurs et ordinateurs Active Directory (ADUC), trois Unités d'Organisation sont créées sous la racine du domaine. Les comptes étudiants sont créés dans OU=ETUDIANTS et ajoutés au groupe de sécurité GRP_ETUDIANTS.

PowerShell Création des OUs et d'un compte étudiant
# Création des Unités d'Organisation
PS C:\> New-ADOrganizationalUnit -Name "ETUDIANTS" -Path "DC=campus,DC=lan"
PS C:\> New-ADOrganizationalUnit -Name "FORMATEURS" -Path "DC=campus,DC=lan"
PS C:\> New-ADOrganizationalUnit -Name "ADMINISTRATION" -Path "DC=campus,DC=lan"
# Création d'un compte étudiant
PS C:\> New-ADUser -Name "Chevalier.Thomas" `
-Path "OU=ETUDIANTS,DC=campus,DC=lan" `
-AccountPassword (ConvertTo-SecureString "Campus@2026!" -AsPlainText -Force) `
-Enabled $true
# Ajout au groupe de sécurité
PS C:\> Add-ADGroupMember -Identity "GRP_ETUDIANTS" -Members "Chevalier.Thomas"
✓ Compte créé et ajouté au groupe GRP_ETUDIANTS
3.3 — Jonction des postes clients au domaine

Sur chaque poste client Windows 10/11 Pro, la jonction au domaine est effectuée via Paramètres → Système → À propos → Domaine. Après redémarrage, le poste est visible dans le conteneur Computers de l'ADUC.

PowerShell Jonction au domaine (depuis le client)
# Jonction au domaine campus.lan
PS C:\> Add-Computer -DomainName "campus.lan" `
-Credential (Get-Credential) `
-Restart
# Vérification post-jonction
PS C:\> (Get-WmiObject -Class Win32_ComputerSystem).Domain
campus.lan
✓ Poste joint au domaine campus.lan avec succès
04Configuration des GPO
4.1 — Création et liaison des GPO

Les stratégies de groupe sont créées depuis la console Gestion des stratégies de groupe (GPMC) et liées à OU=ETUDIANTS. Par mécanisme d'héritage, toutes les GPO liées s'appliquent automatiquement à chaque compte placé dans cette OU.

PowerShell Création et liaison des GPO pédagogiques
# Créer la GPO de restrictions pédagogiques
PS C:\> New-GPO -Name "GPO_RESTRICTIONS_PEDAGOGIQUES"
# Lier la GPO à l'OU Etudiants
PS C:\> New-GPLink -Name "GPO_RESTRICTIONS_PEDAGOGIQUES" `
-Target "OU=ETUDIANTS,DC=campus,DC=lan"
# Paramètre : bloquer le Panneau de configuration
PS C:\> Set-GPRegistryValue -Name "GPO_RESTRICTIONS_PEDAGOGIQUES" `
-Key "HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer" `
-ValueName "NoControlPanel" -Type DWord -Value 1
# Paramètre : bloquer CMD et PowerShell
PS C:\> Set-GPRegistryValue -Name "GPO_RESTRICTIONS_PEDAGOGIQUES" `
-Key "HKCU\Software\Policies\Microsoft\Windows\System" `
-ValueName "DisableCMD" -Type DWord -Value 1
✓ GPO_RESTRICTIONS_PEDAGOGIQUES créée et liée à OU=ETUDIANTS
4.2 — Mappage du lecteur réseau S: (Devoirs)

Le lecteur réseau S: est mappé automatiquement via la GPO GPO_LECTEUR_DEVOIRS configurée dans Configuration utilisateur → Préférences → Paramètres Windows → Mappages de lecteurs. Aucune action manuelle n'est requise de la part de l'étudiant.

PowerShell GPO Mappage Lecteur Réseau S:
# Créer la GPO de mappage lecteur
PS C:\> New-GPO -Name "GPO_LECTEUR_DEVOIRS"
PS C:\> New-GPLink -Name "GPO_LECTEUR_DEVOIRS" -Target "OU=ETUDIANTS,DC=campus,DC=lan"
# Configuration via GPMC : Préférences > Mappages de lecteurs
# Lecteur : S: → \\SRV-AD\Devoirs
# Action : Mettre à jour (Update)
# Ciblage : GRP_ETUDIANTS uniquement
# Vérification côté client
C:\Users\Chevalier.Thomas> net use
OK S: \\SRV-AD\Devoirs Microsoft Windows Network
✓ Lecteur S: mappé sur \\SRV-AD\Devoirs — GPO active
4.3 — GPO Fond d'écran imposé
PowerShell GPO Fond d'écran bureau — Chemin SYSVOL
# Paramètre Clé de registre — Fond d'écran imposé
PS C:\> Set-GPRegistryValue -Name "GPO_FOND_ECRAN" `
-Key "HKCU\Control Panel\Desktop" `
-ValueName "Wallpaper" -Type String `
-Value "\\SRVAD\sysvol\campus.lan\Policies\fond_campus.jpg"
# Empêcher la modification par l'utilisateur
PS C:\> Set-GPRegistryValue -Name "GPO_FOND_ECRAN" `
-Key "HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\ActiveDesktop" `
-ValueName "NoChangingWallPaper" -Type DWord -Value 1
✓ Fond d'écran Campus Numérique imposé — modifications bloquées
05Validation & Tests
5.1 — Mise à jour des GPO et vérification

Après connexion avec un compte étudiant CAMPUS\Chevalier.Thomas, l'authentification se fait auprès du DC SRV-AD via le protocole Kerberos. Les GPO liées à OU=ETUDIANTS sont appliquées pendant l'ouverture de session.

 Terminal interactif — Tests en direct
Windows PowerShell — Poste Client CAMPUS\Chevalier.Thomas  ▮
5.2 — Preuve d'application des restrictions GPO

La validation finale consiste à tenter d'accéder au Panneau de configuration depuis le compte étudiant. La GPO NoControlPanel=1 intercepte la requête et affiche le message d'erreur système « Restrictions » — preuve irréfutable de l'application correcte de la stratégie de groupe.

Restrictions
Boîte de dialogue Windows
Cette opération a été annulée en raison de restrictions en vigueur sur cet ordinateur.
Veuillez contacter votre administrateur système.
Message déclenché lors de l'accès à Panneau de configuration — GPO NoControlPanel=1 actif

06Conclusion & Bilan de Migration
 Migration réussie — Active Directory Opérationnel

Le déploiement d'Active Directory Domain Services sur le serveur Windows Server 2022 du Campus Numérique constitue une transformation radicale et maîtrisée de l'infrastructure de gestion des identités. Les 200 comptes étudiants sont désormais centralisés dans un annuaire LDAP unique, accessibles depuis n'importe quel poste du domaine campus.lan.
